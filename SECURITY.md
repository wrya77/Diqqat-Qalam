# سياسة الأمان — Security Policy

## الإبلاغ عن ثغرة
إن اكتشفت ثغرة أمنية، **لا تفتح Issue عمومياً**. أبلغ مباشرةً عبر:

- البريد: **wrya.design@gmail.com** (الموضوع: `SECURITY`)

نسعى للردّ خلال 72 ساعة، وتنسيق إفصاح مسؤول قبل أي نشر علني.

## النطاق
يشمل التطبيق طبقات حسّاسة (مصادقة Supabase، مدفوعات، اتصال CNC). الأمور ذات الأولوية:
- تجاوز المصادقة أو التصعيد في الصلاحيات.
- ثغرات الدفع/الاشتراك (تجاوز السعر، تكرار المعاملات).
- SSRF / اجتياز المسارات / حقن في مسارات الخادم.
- كشف الأسرار أو بيانات المستخدمين.

## ما هو خارج النطاق
- هجمات الحرمان من الخدمة الحجمية (DoS) عبر الإغراق.
- الإبلاغ الآلي دون إثبات مفهوم (PoC) قابل لإعادة الإنتاج.

## ممارسات المستودع
- الأسرار في `.env` فقط (محجوبة عبر `.gitignore`)؛ `.env.example` يوثّق المتغيرات دون قيم.
- فحص `npm audit --audit-level=high` يعمل في CI على كل دفع/طلب دمج.
- تحديثات التبعيات عبر Dependabot.
